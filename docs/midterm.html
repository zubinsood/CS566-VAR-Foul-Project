Mid-Term Report

Current Progress

Since submitting our proposal, we have focused on understanding the SoccerNet MV-Foul dataset and setting up the baseline multi-view model provided by the challenge organizers. The dataset includes short, multi-angle video clips labeled with both foul severity (No Card, Yellow, Red, etc.) and foul type (e.g., Pushing, Holding).

We have successfully:

  • Reviewed the dataset structure and verified the annotation format.
  • Cleaned/verified the existing dataset, ensuring that the clips are accurate.
  • Configured our development environment (Github, PyTorch, TorchVision, and SoccerNet utilities).
  • Implemented and tested the baseline model, MVNetwork, which uses a 3D convolutional backbone (R(2+1)D-18, S3D, or MViT-V2-S) combined with a multi-view aggregation layer (MVAggregate).
  • Validated that the model can process dummy multi-view input clips and produce consistent output shapes, confirming that the architecture and dependencies are functioning correctly.

At this stage, we are preparing the dataset preprocessing pipeline and expect to begin baseline training during the first week of November.

Challenges

  • Dataset Scale and Access: The SoccerNet MV-Foul dataset is large (tens of GBs/3000+ videos), and downloading and organizing the synchronized multi-view videos took longer than expected.
  • Baseline Code Complexity: The official repository combines multiple tasks (foul detection, spotting, and classification), which made it challenging to isolate/understand the actual logic that relates to the foul detection.

Changes

Our original plan included both multi-view model and pose-based features. Based on current progress, we are slightly adjusting priorities, focus mainly on improving the multi-view model until we get a decent evaluation score and then modify it later.

Evaluation

We will evaluate our models using the official SoccerNet MV-Foul recognition metrics, which are integrated directly into the SoccerNet Python package and supported by the EvalAI evaluation server.

1. Foul Severity (4 classes)

  • No Offence
  • Offence + No Card
  • Offence + Yellow Card
  • Offence + Red Card

The evaluation metric for this task is balanced accuracy, which accounts for class imbalance.

2. Foul Type (8 classes)

  • Tackling, Standing Tackling, High Leg, Holding, Pushing, Elbowing, Challenge, and Dive.

This task also uses balanced accuracy as the primary metric.
